Hugo

Hugo is a static site generator developed in Go. You write content using plain markdown files, and Hugo build the website following a template defined by you (or by some theme from the Internet).

Inside each template file, you have access to the Hugo's template language that allow you to structure the markdown content over the HTML structure.

Installing Hugo#

In macOS, you can install Hugo using Homebrew.

brew update
brew install hugo

Creating a new site#

Use the following command to start a new site:

hugo new site my-site
cd my-site

Adding a theme#

You need template files to be able to show your markdown content as HTML webpage. You can download a theme from the internet or try to build your own.

Lets use the PaperMod theme for this example. We can just clone the theme repository to themes/<theme-name>:

git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

Then, update the hugo.yaml file to include the theme:

theme: ["PaperMod"]

Creating New Post#

You can create a new post manually or using Hugo CLI. To create manually, just create the .md file, and manually write the front matter. To create a new post using the CLI, use the following command:

hugo new content content/posts/post-1.md

You can also create folders to group multiple markdown files.

hugo new content content/posts/big-post/index.md

Troubleshooting#

When things go wrong, try the following commands to clean cache.

hugo --cleanDestinationDir
hugo server --disableFastRender
rm -rf ./public && hugo server  --panicOnWarning --ignoreCache --disableFastRender --cleanDestinationDir

Serving Locally#

You can run hugo locally with a simple hugo server command. Use --buildDrafts flag to include posts with draft: true in the front matter.

hugo server --buildDrafts

Addons#

Overriding Theme Files#

You can overwrite a theme feature by copying the file from the theme folder to the layout folder. Hugo will prioritize the file inside the layout folder. For example, lets say you want to modify the index.html file, you can copy the file from themes/foo/layouts/index.html and paste it in layouts/index.html.

See Override a Hugo Theme for more information.

Partials#

Hugo Partial is a piece of a layout (HTML) that lives in its own file. You can import the partial from the layout itself. For example, the PaperMod theme imports several partials in its index file like headers.html.

Shortcodes#

A Hugo Shortcode is a template content that is invoked from the markup file.

Some shortcodes are built-in like figure which allow us to insert images in your markdown content.

Modifying CSS#

If you want to modify some CSS like to adjust h1 size or font, you can modify the header.html partial of the PaperMod theme to include a custom CSS file which will overwride a specific CSS selector.

Create the custom.css file in the static folder:

/static/custom.css

Add a new entry into hugo.yaml

params:
  custom_css: ["custom.css"]

Modify the header partial to import the file. You can modify the theme file directly at themes/PaperMod/layouts/partials/header.html, or copying it to the layout folder keep the original theme file intact.

EDIT: you should copy extended_head.html from themes to layout/partials, since the PaperMod is already including it from inside header.html file.

{{ range .Site.Params.custom_css -}}
    <link rel="stylesheet" href="{{ . | absURL }}">
{{- end }}

Custom Code Syntax Highlight#

hugo gen chromastyles –style=monokailight > themes/PaperMod/assets/css/includes/chroma-styles.css

Adding Google Analytics#

To add analytics to your Hugo website, assuming your current theme doesn’t support it automatically by just updating the settings in hugo.yaml, do the following:

  • Go to analytics.google.com, crete your account, and copy the code provided by Google.
  • Create a new partial file called analytics in hugo_root/layouts/_partials/google/analytics.html
  • Paste the code there, save, close
  • Include the partial in your theme. For PaperMod, edit the file themes/PaperMod/layouts/partials/head.html to include the created partial using {{ - partial "google/analytics.html" . -}}.
