Fractals

It is facinating how we can produce beautiful visuals by just checking if the the multiplication and sum of a complex number is stable or not.

For each pixel x,y of the canvas, we start with a complex number z=(0,0) and keep multiplying z by itself and adding the point x,y.

z' = z * z + p

or

z_n+1 = z_n * z_n + p

We keep doing this for N times, normally 100 is enough for a good resolution.

If after a certain number of iterations the computation is still stable (e.g. the absolute value of the complex number is less than a certain threshold), we can say that the point is in the set. Otherwise, the point is not in the set.

Finally we can define a color for each point based on the number of iterations it took to become unstable, for example, linear map the number of iteration to HUE domain [0,360) in the HSV color space.

Mapping#

For each point of the canvas <x,y> that goes from <0,0> to <w,h>, we need to map it to the domain of the complex plane. We can start maping x and y from [0,w] to [-1,+1].

If we want a “zoomed” fractal, we can decrease the domain of the complex number, for example, by mapping x and y from [0,w] to [-0.5,+0.5].

The middle point of the canvas, at x = w/2 and y = h/2 is mapped to point 0,0 at the complex plane.

Example#

Lets say we have a canvas with w=200,h=200 and we are mapping to the complex plane [-1,+1].

Lets test if the point x=50,y=50 is at the set or not.

Pixel \( (50,50) \) maps to complex coordinate ((-0.5, 0.5)).
